# 前端界面说明

## 文件结构

```
frontend/
  index.html    —— 流量监控 + 防火墙配置 Web 界面（单页双 Tab）
  README.md     —— 本文件
```

## 使用方法

1. 确保后端 `server.py` 已在对应机器上运行（默认监听 `localhost:8080`）
2. 直接用浏览器打开 `frontend/index.html`
3. 如果后端运行在其他地址，可通过 URL 参数指定：
   ```
   file:///C:/.../index.html?api=http://192.168.6.100:8080
   ```

## 功能说明

### 流量监控 Tab

| 功能 | 说明 |
|------|------|
| 概览统计卡片 | 显示活跃连接数、累计总流量、全局峰值速率、最后更新时间 |
| 实时数据表格 | 每 2 秒自动刷新，展示每条流的源/目的 IP、累计流量、2s/10s/40s 速率、历史峰值 |
| 方向标记 | 自动识别出站（OpenWrt → 外部）、入站（外部 → OpenWrt）、本地通信 |
| 速率趋势图 | Chart.js 折线图，同时显示 2s/10s/40s 三种粒度的速率变化，保留最近 60 个数据点 |
| 暂停/恢复 | 可暂停自动刷新以仔细查看数据，也可手动点击立即刷新 |

### 防火墙配置 Tab

| 功能 | 说明 |
|------|------|
| 添加规则 | 表单输入：协议类型（TCP/UDP/ICMP）、源/目标 IP、端口号、处理动作（ACCEPT/DROP/REJECT） |
| 前端校验 | IP 地址格式验证、必填项检查，防止无效数据提交 |
| 规则列表 | 展示当前所有防火墙规则，支持单条删除 |
| 操作反馈 | 成功/失败消息提示，8 秒后自动消失 |

## API 依赖

前端依赖后端提供的 4 个 API 接口（详见项目根目录 `前端API文档.md`）：

- `GET /api/stats` — 流量数据
- `GET /api/firewall/list` — 规则列表
- `POST /api/firewall/add` — 添加规则
- `POST /api/firewall/del` — 删除规则

## 技术栈

- 纯原生 HTML/CSS/JavaScript，无框架依赖
- Chart.js 4.4（CDN 加载，用于速率趋势图；国内 CDN 优先，jsdelivr 备用）
- 响应式布局，适配桌面和移动端
- 跨域请求支持（后端已启用 CORS）

## 注意事项

- 如果 CDN 加载 Chart.js 失败，图表区域将无法显示，但表格和其他功能不受影响
- 图表数据点在切换选中的流时会清空重新累积
- 防火墙规则的实际生效需要 OpenWrt 上的 `firewall.sh` 脚本支持（由其他组员负责）
