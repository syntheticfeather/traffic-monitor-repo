## 测试目的

验证流量监控 Web 前端能够：
1. 正确从后端 API 获取实时流量数据
2. 统计卡片（活跃连接数、累计总流量、全局峰值速率）正确更新
3. **流量速率趋势图（Chart.js 折线图）能实时绘制 2s/10s/40s 三条速率曲线**
4. 数据表格正确展示按速率排序的流记录
5. 每 2 秒自动刷新，图表持续累积数据点

## 测试环境架构

```
OpenWrt VM (192.168.6.100)
  ├── monitor (C 程序) — libpcap 抓包 → stats.txt
  ├── 持续 ICMP Ping 流量 (8.8.8.8, 223.5.5.5)
  └── firewall.sh
         │ SSH
         ▼
Windows 宿主机
  ├── server.py :8080 — 读 stats.txt + 防火墙 API
  └── 前端 HTML — 轮询 /api/stats，渲染图表
```

## 测试步骤

### 1. 启动后端服务

```bash
# OpenWrt: 启动流量监控
ssh root@192.168.6.100 './mnt/p0/traffic_monitor/monitor &'

# OpenWrt: 生成持续测试流量（多路并发 Ping）
ssh root@192.168.6.100 '
  for i in $(seq 1 8); do
    ping -c 200 -i 0.2 8.8.8.8 &
    ping -c 200 -i 0.2 223.5.5.5 &
  done
'

# Windows: 启动后端 API
python src/server.py  # 端口 8080
```

### 2. 打开前端页面

浏览器打开 `http://localhost:8089/index.html?api=http://localhost:8080`

### 3. 观察并记录

- 每 2 秒自动刷新一次
- 累计观察 **64 秒**，收集 **32 个数据点**
- 截图记录页面状态

## 已知事项

1. **Chart.js CDN 加载**：在无互联网访问的隔离环境中，Chart.js 需本地部署。本项目已提供本地 `chart.umd.min.js` 备选方案。
2. **速率波动**：ICMP Ping 为固定间隔发包，各时间窗口内包数量波动属正常网络行为。
3. **端口占用**：若 8080 端口被占用，可通过环境变量 `PORT=8081` 启动 server.py，前端通过 `?api=http://localhost:8081` 参数适配。

## 测试结论
