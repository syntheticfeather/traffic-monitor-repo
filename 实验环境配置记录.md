# 计算机网络实验二 — 实验环境配置记录

## 一、OpenWrt 虚拟机部署

### 1.1 镜像下载与转换

- 下载地址：`https://downloads.openwrt.org/releases/24.10.0/targets/x86/64/`
- 选择文件：`generic-ext4-combined-efi.img.gz`
  - ext4：可读写文件系统，便于后续安装 samba 等软件
  - efi：兼容 VMware UEFI 引导
  - combined：内核 + 根文件系统打包，可直接启动
- 格式转换：指导书中的V2V不可用，使用 `qemu-img` 将 `.img` 转为 `.vmdk`

```bash
qemu-img.exe convert -f raw -O vmdk generic-ext4-combined-efi.img openwrt.vmdk
```

### 1.2 VMware 虚拟机创建

| 配置项 | 值 |
|--------|-----|
| 操作系统 | Linux → 其他 Linux 5.x 64 位 |
| 内存 | 1024 MB |
| 处理器 | 1 核 |
| 网络 | NAT 模式 |
| 磁盘 | 使用现有虚拟磁盘（vmdk） |

## 二、网络配置

### 2.1 故障排查过程

**现象**：OpenWrt ping 宿主机不通。

**排查步骤**：
1. 初始状态 /etc/config/network 中为空，先填入基本信息
2. `ip addr show` — 确认网卡名称为 `eth0`，状态 UP
3. Windows 端 `ipconfig` — VMnet8 网段为 `192.168.6.0/24`
4. 在 VMware 虚拟网络编辑器中确认 NAT 子网为 `192.168.6.0`

**根因**：网关配置错误。

- ❌ `gateway '192.168.6.1'` — 这是宿主机在虚拟网络中的地址，不负责转发
- ✅ `gateway '192.168.6.2'` — 这是 VMware NAT 服务的内置虚拟网关

```
OpenWrt(192.168.6.100) → NAT网关(192.168.6.2) → 宿主机真实网卡 → 外网
             宿主机 VMnet8(192.168.6.1) — 仅用于通信，不做转发
```

### 2.2 最终配置

`/etc/config/network`：

```bash
config interface 'loopback'
        option ifname 'lo'
        option proto 'static'
        option ipaddr '127.0.0.1'
        option netmask '255.0.0.0'

config interface 'lan'
        option ifname 'eth0'
        option proto 'static'
        option ipaddr '192.168.6.100'
        option netmask '255.255.255.0'
        option gateway '192.168.6.2' # 此处从 6.1 修改为 6.2
        list dns '8.8.8.8'
        list dns '114.114.114.114'
```

配置生效：
```bash
/etc/init.d/network restart
```

验证：
```bash
ping -c 3 192.168.6.2   # NAT 网关通
ping -c 3 8.8.8.8       # 外网通
```

## 四、Samba 文件共享

### 4.1 安装

```bash
opkg update
opkg install luci-app-samba4
```

### 4.2 配置

`/etc/config/samba4` 添加共享：

```
config sambashare
        option name 'share'
        option path '/mnt/p0'
        option read_only 'no'
        option guest_ok 'yes'
        option create_mask '0777'
        option dir_mask '0777'
        option browseable 'yes'
```

### 4.3 启用

```bash
mkdir -p /mnt/p0
chmod -R 777 /mnt/p0
/etc/init.d/samba4 enable
/etc/init.d/samba4 restart
```

Windows 访问：`Win+R` → `\\192.168.6.100` → 看到 `share` 目录。

## 五、交叉编译环境

### 5.1 核心问题

OpenWrt 24.10 使用 **musl libc**，而 WSL (Ubuntu) 使用 **glibc**。两者 ABI 不兼容：
- glibc 编译的二进制 → OpenWrt 上报 `symbol not found` 或 `ld-linux-x86-64.so.2: not found`
- 动态链接的 `.so` 库也不能直接混用

### 5.2 失败的尝试

| # | 方案 | 失败原因 |
|---|------|---------|
| 1 | `gcc -static`（WSL glibc） | libpcap 静态库拖入 dbus/rdma/infiniband 等大量依赖，无法链接 |
| 2 | 动态编译 + 拷 `libpcap.so` 到 OpenWrt | glibc 的 .so ≠ musl 的 .so，ABI 不兼容 |
| 3 | OpenWrt 本地编译（`opkg install gcc`） | overlay 仅 30MB，装不下 gcc |
| 4 | 扩磁盘 + 本地编译 | `fdisk` 手动交互搞坏 GPT 备份表，VM 变砖；`parted`/`gdisk` 反复修复失败 |
| 5 | `musl-gcc` 交叉编译 | 缺内核头文件，`./configure` 报 `No supported packet capture interface` |
| 6 | OpenWrt SDK 交叉编译 | SDK 不带 feeds 和 toolchain，需要从源码构建整个工具链 |
| 7 | Docker Alpine（musl） | WSL 没有 Docker，安装耗时 |

### 5.3 最终方案：OpenWrt Toolchain 交叉编译

**关键区别**：下载 **Toolchain**（约 100MB），而非 SDK（约 300MB）。Toolchain 自带 `x86_64-openwrt-linux-musl-gcc` 和 musl 库。

**步骤**：

1. 下载 OpenWrt Toolchain：
```bash
wget https://downloads.openwrt.org/releases/24.10.0/targets/x86/64/openwrt-toolchain-24.10.0-x86-64_gcc-13.3.0_musl.Linux-x86_64.tar.zst
tar --zstd -xf openwrt-toolchain-24.10.0-x86-64_gcc-13.3.0_musl.Linux-x86_64.tar.zst
```

2. 下载 libpcap 源码（**必须用 tcpdump.org 官方版**，GitHub 版缺 `./configure`）：
```bash
wget https://www.tcpdump.org/release/libpcap-1.10.5.tar.gz
tar xf libpcap-1.10.5.tar.gz
```

3. 用 Toolchain 交叉编译 libpcap（需提前 `sudo apt install bison`）：
```bash
T=/path/to/toolchain-x86_64_gcc-13.3.0_musl
export PATH="$T/bin:/usr/bin:/bin"
export STAGING_DIR="$T"
cd libpcap-1.10.5
./configure --host=x86_64-openwrt-linux-musl --prefix="$T" --disable-shared --disable-dbus
make -j$(nproc)
make install
```

4. 编译监控程序（静态链接 musl + libpcap）：
```bash
x86_64-openwrt-linux-musl-gcc -static -o monitor monitor.c hash.c -lpcap -lpthread
```

输出：`monitor: ELF 64-bit LSB executable, x86-64, version 1 (SYSV), statically linked`

5. 快速重编脚本 [`compile.sh`](../compile.sh)：libpcap 只需编译一次，后续改代码只执行第 4 步。

6. 上传到 OpenWrt：
```bash
cat monitor | wsl -e bash -c "ssh root@192.168.6.100 'cat > /mnt/p0/traffic_monitor/monitor && chmod +x /mnt/p0/traffic_monitor/monitor'"
```

## 六、关键踩坑备忘

| 问题 | 原因 | 解决 |
|------|------|------|
| qemu-img Windows 用法 | 需带 `.exe` 后缀 | `qemu-img.exe convert ...` |
| firstboot 后 network 清空 | firstboot 恢复出厂 | 手动重写 `/etc/config/network` |
| ping 全部 loss | 网关配成宿主机 IP 而非 NAT 网关 | gateway 改为 `192.168.6.2` |
| overlay 空间不足 | 镜像文件仅 ~100MB | 走交叉编译，不在 OpenWrt 上编译 |
| glibc vs musl ABI | WSL glibc 与 OpenWrt musl 不兼容 | OpenWrt Toolchain 交叉编译 |
| GitHub libpcap 无 configure | GitHub 发布包用 cmake | 用 tcpdump.org 官方 tarball |
| configure 缺 bison | libpcap 构建需要 yacc | `sudo apt install bison` |
| `Program Files (x86)` 在 PATH | 括号导致 bash 语法错误 | PATH 只设 `/usr/bin:/bin` |

## 七、流量监控程序 — 数据结构设计

### 7.1 哈希表

| 决策项 | 方案 |
|--------|------|
| 键 | (src_IP 高 32 位 \| dst_IP 低 32 位) → mod 槽数 |
| 哈希函数 | 64 位整数 mod 1999（质数） |
| 碰撞处理 | 链式存储（每个槽挂链表） |
| 结点分配 | 预分配 `pool[MAX_FLOWS]` 数组，不用 `malloc` |
| 满时淘汰 | FIFO，踢最早建立的流记录 |

### 7.2 流结点字段

| 字段 | 含义 |
|------|------|
| `src_ip` | 源 IP（键，同时用于展示） |
| `dst_ip` | 目的 IP（键，同时用于展示） |
| `total_bytes` | 累计字节数 |
| `max_rate` | 历史峰值速率（B/s） |
| `last_bytes` | 上次采样时的累计值（用于算当前瞬时速率） |
| `first_seen` | 首次出现时间戳（avg = total / 时长） |

### 7.3 架构图

```
主线程（main）
  ├── 抓包线程（capture_thread）
  │     ├── pcap_open_live("eth0")
  │     ├── pcap_setfilter("ip")
  │     └── pcap_loop → packet_handler()
  │           └── 解析 IP 头 → 找/建流结点 → 更新 total_bytes, max_rate
  │
  └── 统计线程（stats_thread）
        └── 每 2s：
              ├── 遍历所有流
              ├── 算瞬时速率：(now_bytes - last_bytes) / 2
              ├── 更新 max_rate
              └── 写 stats.txt
```
